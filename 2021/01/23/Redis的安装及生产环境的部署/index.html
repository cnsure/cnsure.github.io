Redis的安装以及生产环境的部署

一 环境准备

1. 首先安装虚拟机及管理工具

参考Vagrant虚拟机管理，当然也可以使用自己的虚拟机直接操作，这样的话可以忽略1、2步

2. 管理虚拟机

创建如下的目录结构，并配置Vagrantfile

# -*- mode: ruby -*-
# vi: set ft=ruby :

# 只列出了必要的配置，需要分别配置network、hostname 完成4个虚拟机的配置
Vagrant.configure("2") do |config|
  config.vm.box = "centos/7"
  config.vm.hostname = "vagrant01"
  config.vm.network "private_network", ip: "192.168.56.101"
  config.vm.synced_folder "../data", "/vagrant_data"
  config.vm.provider "virtualbox" do |vb|
    # Display the VirtualBox GUI when booting the machine
    # vb.gui = true
    vb.name = "vagrant01"
    # Customize the amount of memory on the VM:
    vb.memory = "1024"
  end

end

3. 配置host文件

192.168.56.101 vagrant01
192.168.56.102 vagrant02
192.168.56.103 vagrant03
192.168.56.104 vagrant04

4. 配置服务间的免密登录

本配置都是基于root用户

  1. 生成秘钥ssh-keygen -t rsa然后一路回车
  2. 会在/root/.ssh目录下生成秘钥文件
  3. cp id_rsa.pub authorized_keys将公钥复制为authorized_keys文件，此时使用ssh连接本机就不需要输入密码了
  4. 使用ssh-copy-id -i hostname命令将本机的公钥拷贝到指定机器的authorized_keys文件中,这样其他机器连接本机就不需要密码了

注意使用vagrant用户时，默认有authorized_keys文件，这个vagrant ssh时需要的，所以不要直接覆盖（导致vagrant ssh 连接不上），可以 cat id_rsa.pub >> authorized_keys追加

5. 安装perl

使用redis的lua脚本时会用到

yum install -y gcc
# 下载
wget http://www.cpan.org/src/5.0/perl-5.16.1.tar.gz
# 解压
tar -xzf perl-5.16.1.tar.gz
cd perl-5.16.1
# 配置
./Configure -des -Dprefix=/usr/local/perl
# 安装
make && make test && make install
# 校验
perl -v

6. 安装tcl8.6.1

tar -zxvf tcl8.6.1-src.tar.gz
cd tcl8.6.1/unix/
./configure
make && make test && make install

二 单机redis安装

1. 安装redis

tar -zxvf redis-3.2.8.tar.gz
cd redis-3.2.8/
make && make install

2. 安装redis服务

使用redis提供的util来安装redis的后台服务

cd utils
./install_server.sh
# 然后一路回车，使用默认配置

3. 启动及查看redis服务

# 启动redis服务
systemctl start redis_6379.service
# 查看redis服务状态
systemctl status redis_6379.service

将3台服务搭建好做好准备工作

4. redis配置文件

# 绑定ip 可以远程访问
bind 127.0.0.1 192.168.56.101 

# 指定端口号
port 6379

# 后台运行
daemonize yes

# 指定pid文件
pidfile /var/run/redis_6379.pid

# 日志路径
logfile /var/log/redis_6379.log

# 指定密码
requirepass redis

# RDB 的存储策略
save 900 1
save 300 10
save 60 10000

# 数据存储目录
dir /var/lib/redis/6379

# aof文件存储
appendonly yes

# aof文件存储文件
appendfilename "appendonly.aof"

# aof存储策略
appendfsync everysec

# aof文件重写策略
auto-aof-rewrite-percentage 100
auto-aof-rewrite-min-size 64mb
